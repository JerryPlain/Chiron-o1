### 2.5 项目内 PyTorch forward 示例（以 `qwen_vl` 推理为例）

说明：
- 你项目里 `src/qwenvl_forward.py` 暴露的是 `qwenvl_forward(...)` 包装函数。
- 代码表面调用 `model.generate(...)`，但底层每一步解码都会走模型的 `forward(...)`。
- 面试时可以按下面伪代码讲“多模态输入 -> tensor 化 -> forward/generate -> decode”。
- 先记一句总链路：`question + image_paths` 先组织成 `messages`（对话协议），再交给 `processor` 变成 PyTorch tensors，最后模型基于 tensors 做自回归生成。

你问的几个“为什么”先给结论：
- 为什么用 `messages`：因为 Qwen-VL 是 chat 风格接口，输入协议不是单个字符串，而是 `[{role, content}, ...]`。
- 为什么 `user.content` 一开始是空的：因为它要装“多个内容块”（多张图 + 文本），所以先 `[]` 再逐个 `append`。
- 为什么不像直接传 `question`：`question/prompt` 本质没变，只是被包装成 `user.content` 里的 `{"type":"text","text": prompt}`。
- 为什么用 `processor`：它负责统一做 tokenizer + image preprocess + padding + mask + tensor 化，这是模型可执行输入的标准入口。
- 为什么 `process_vision_info`：把 message 里的 `image/video` 条目抽取并整理成 processor 所需格式。
- 为什么变 tensor：PyTorch 模型只吃数值张量，不吃 Python 字符串/路径。`return_tensors="pt"` 就是在做这件事。

```pseudo
function qwenvl_forward(model, processor, prompt, img_paths, temperature):
    # [输入]
    # prompt    = 你的问题文本（语义上就是 question）
    # img_paths = 图片路径列表

    # 1) 组装 chat 协议消息（不是随便写，模型就是按这个协议训练的）
    messages = [
        {"role": "system", "content": CLINICAL_SYSTEM_PROMPT},
        {"role": "user", "content": []}
    ]

    # user.content 为什么是空列表：
    # 因为它要容纳多个片段（N 张图 + 1 段文本），顺序很重要。
    # 这里先 append 图，再 append 文本，表示“先看图，再看问题”。
    for p in img_paths:
        messages[1]["content"].append({"type": "image", "image": p})

    # 你的 question/prompt 在这里进入消息体
    messages[1]["content"].append({"type": "text", "text": prompt})

    # 2) 把 messages 转成模型前处理可接受的中间表示
    # apply_chat_template: 把 role/content 序列化成模型的对话模板文本
    text_prompt = processor.apply_chat_template(
        messages, tokenize=false, add_generation_prompt=true
    )

    # process_vision_info: 从 messages 中提取视觉项并整理结构
    # 输出 image_inputs/video_inputs 仍是“待 processor 处理”的中间形态，不是最终 tensor
    (image_inputs, video_inputs) = process_vision_info(messages)

    # 3) 统一 tensor 化（关键步骤）
    # processor 在这里同时做：
    # - 文本 tokenizer -> input_ids
    # - 图像预处理 -> pixel_values
    # - padding/mask 构建
    # - 打包为 BatchEncoding
    inputs = processor(
        text=[text_prompt],
        images=image_inputs, 
        videos=video_inputs,
        padding=true,
        return_tensors="pt"
    ).to(model.device)
    # return_tensors="pt" => 全部变为 PyTorch tensor
    # .to(model.device)   => 放到和模型相同设备（CPU/GPU）

    # 4) 生成（外层调用 generate，内层每步都在 forward）
    # 可口述为：generate = 多次 forward + 采样/贪心解码
    generated_ids = model.generate(
        **inputs,
        max_new_tokens=2048,
        repetition_penalty=1.0,
        temperature=temperature
    )

    # 5) 裁掉“输入部分 token”，只保留“新生成 token”
    # out_ids = [输入token + 新token]，所以需要按输入长度截断
    trimmed = []
    for (in_ids, out_ids) in zip(inputs.input_ids, generated_ids):
        trimmed.append(out_ids[len(in_ids):])

    # 6) token ids -> 字符串
    output = processor.batch_decode(
        trimmed,
        skip_special_tokens=true,
        clean_up_tokenization_spaces=false
    )[0]
    return output
```

### 2.6 对照示例：`internvl_forward`（和 Qwen-VL 有明显不同）

先看差异结论（面试可直接说）：
- Qwen-VL：输入先组织成 `messages`，再用 `processor` 一次性做 text+vision 打包。
- InternVL：不走 `messages` 协议，先手工把每张图切 patch 并归一化，再调用 `model.chat(...)`。
- Qwen-VL 的视觉处理更多封装在 `processor/process_vision_info`；InternVL 的视觉处理在你自己代码里（`dynamic_preprocess/load_image`）。
- InternVL 多图时要额外传 `num_patches_list`，告诉模型每张图对应多少 patch。

```pseudo
function internvl_forward(model, tokenizer, prompt, img_paths, temperature):
    # [输入]
    # prompt    = 问题文本（这里直接传给 model.chat，不包 messages）
    # img_paths = 图片路径列表

    # 1) 把每张图转成 patch tensor（这是和 Qwen-VL 最大差异）
    all_pixel_values = []
    num_patches_list = []

    for p in img_paths:
        # load_image 内部做了：
        # - RGB 转换
        # - 按长宽比动态切块（dynamic_preprocess）
        # - 每块 resize + normalize
        # - stack 成 [num_patches, 3, H, W]
        pixel_values = load_image(p, max_num=12)

        # 推理 dtype/device 对齐
        pixel_values = pixel_values.to(bfloat16).to(cuda)

        all_pixel_values.append(pixel_values)
        num_patches_list.append(pixel_values.size(0))

    generation_config = {
        "max_new_tokens": 2048,
        "do_sample": true,
        "temperature": temperature
    }

    # 2) 单图和多图分支
    if len(img_paths) == 1:
        # 单图：直接 chat
        response = model.chat(
            tokenizer,
            all_pixel_values[0],   # shape: [n_patch, 3, H, W]
            prompt,
            generation_config
        )
    else:
        # 多图：先拼接 patch，再告诉模型每张图有多少 patch
        merged_pixel_values = cat(all_pixel_values, dim=0)
        response = model.chat(
            tokenizer,
            merged_pixel_values,   # shape: [sum_patch, 3, H, W]
            prompt,
            generation_config,
            num_patches_list=num_patches_list
        )

    return response
```

最小示意（和 2.5 对照看）：

```text
输入：
- prompt = "What is the most likely diagnosis?"
- img_paths = ["a.png", "b.png"]

中间：
- a.png -> [5, 3, 448, 448]
- b.png -> [7, 3, 448, 448]
- merged_pixel_values -> [12, 3, 448, 448]
- num_patches_list = [5, 7]

输出：
```

- InternVL 是“手工视觉预处理驱动 + chat 接口吃 patch tensor”。

### 3.1 二分查找

```pseudo
function binary_search(nums, target):
    l = 0, r = len(nums)-1
    while l <= r:
        m = l + (r-l)//2
        if nums[m] == target: return m
        if nums[m] < target: l = m+1
        else: r = m-1
    return -1
```

```pseudo
function longest_unique_substring(s):
    count = map()
    left = 0
    ans = 0

    for right in [0..len(s)-1]:
        count[s[right]] += 1
        while count[s[right]] > 1:
            count[s[left]] -= 1
            left += 1
        ans = max(ans, right-left+1)
```

```pseudo
function subarray_sum_k(nums, k):
    freq = {0:1}
    prefix = 0
    ans = 0

    for x in nums:
        prefix += x
        ans += freq.get(prefix-k, 0)
```

```pseudo
function next_greater(nums):
    ans = array(len(nums), -1)
    stack = []  # store index, values decreasing

    for i in [0..len(nums)-1]:
```

### 3.6 BFS（最短步数）

```pseudo
function bfs_shortest(start, target):
    q = queue([start])
    visited = set([start])
    step = 0

    while q not empty:
        size = q.size()
        repeat size times:
            cur = q.pop_front()
            if cur == target: return step
            for nxt in neighbors(cur):
                if nxt not in visited:
                    visited.add(nxt)
                    q.push_back(nxt)
        step += 1

    return -1
```

### 3.7 DFS 回溯（全排列）

```pseudo
function permute(nums):
    ans = []
    used = [false] * len(nums)

    function backtrack(path):
        if len(path) == len(nums):
            ans.append(copy(path))
            return
        for i in [0..len(nums)-1]:
            if used[i]: continue
            used[i] = true
            path.append(nums[i])
            backtrack(path)
            path.pop()
            used[i] = false

    backtrack([])
    return ans
```

### 3.8 并查集（连通分量）

```pseudo
class DSU:
    parent, rank

    function find(x):
        if parent[x] != x:
            parent[x] = find(parent[x])
        return parent[x]

    function union(a, b):
        ra = find(a), rb = find(b)
        if ra == rb: return
        if rank[ra] < rank[rb]: swap(ra, rb)
        parent[rb] = ra
        if rank[ra] == rank[rb]: rank[ra] += 1
```

```pseudo
function knapsack01(weights, values, cap):
    dp = [0] * (cap + 1)
    for i in [0..n-1]:
        for c in [cap..weights[i]] descending:
            dp[c] = max(dp[c], dp[c - weights[i]] + values[i])
    return dp[cap]
```

### 3.11 LRU（哈希表 + 双向链表）

```pseudo
class LRUCache(capacity):
    map = key -> node
    list = doubly_linked_list  # MRU at head

    function get(key):
        if key not in map: return -1
        node = map[key]
        list.move_to_head(node)
        return node.value

    function put(key, value):
        if key in map:
            node = map[key]
            node.value = value
            list.move_to_head(node)
            return
        node = new Node(key, value)
        map[key] = node
        list.add_head(node)
        if size(map) > capacity:
            tail = list.remove_tail()
            delete map[tail.key]
```

## 4. 工程设计高频伪代码

### 4.1 生产者-消费者

```pseudo
queue = bounded_queue()
mutex = lock()
not_empty = condition()
not_full = condition()

function producer(item):
    lock(mutex)
    while queue.is_full(): wait(not_full, mutex)
    queue.push(item)
    signal(not_empty)
    unlock(mutex)

function consumer():
    lock(mutex)
    while queue.is_empty(): wait(not_empty, mutex)
    item = queue.pop()
    signal(not_full)
    unlock(mutex)
    return item
```

### 4.2 限流（令牌桶）

```pseudo
class TokenBucket(rate, capacity):
    tokens = capacity
    last_ts = now()

    function allow(request_n=1):
        refill()
        if tokens >= request_n:
            tokens -= request_n
            return true
        return false

    function refill():
        delta = now() - last_ts
        tokens = min(capacity, tokens + delta * rate)
        last_ts = now()
```
